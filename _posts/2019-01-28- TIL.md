# 백지노트

## 변수

코틀린의 변수는 var(가변 변수)과 val(불변 변수)이 있다. **기본적으로 지역변수이며, 전역 변수의 개념이 없다.**  모든 변수는 **사용하기 전에** 반드시 한 번 초기화되어야 한다. 선언을 할 수는 있으나 선언만 된 변수를 사용하려고 하면 컴파일 에러가 발생한다.

## git 사용법

로컬에서 작업하기 전에 원격 저장소의 내용을 pull해서 가져온다. 원격 저장소에 있는 원본을 내 로컬에 다운받아서 동기화 시킨다고 봐야지. 그 다음에 작업을 하고 나서 다시 나의 로컬을 원격에 push 한다. push 하기 위해서는 일단 변경사항을 add하고 (`*`로 하던지 `파일이름`으로 하던지) commit 한 다음 push로 올려준다.

init은 깃을 초기화 시킬 때 .. 혹은 뭔가가 꼬였다고 생각될 때 싹 날리고 다시 시작한다는 느낌? 

branch는 블로그에서는 사용할 필요가 없을 것 같다. 굳이 맥과 윈도우 작업을 나누어 브랜치할 필요가....있을까? 다만 코틀린 작업의 경우에는 브랜치를 사용해보도록 하자.

## Micro:Bit Genius Kit

간만에 마빗을 다뤄보았다. 엣지 확장센서를 사용하면 마이크로 비트의 다양한 핀을 바로바로 사용할 수 있어서 좋다. 오늘 테스트 해본 바로는 확장센서와 다양한 센서를 달아줄 케이스가 필요해 보인다. 이는 알리에서 Lepin을 사다가 해결해 볼까... 하는 생각이 든다.

- LED : 특이 사항 없다. P1에서 잘 작동한다. 
- 충돌센서 : 눌렸다 떨어져야 작동함. P1에서는 선명하게 작동하나 이외의 핀에서는 그렇지 않다. 핀마다 전용 기능이 따로 배정되어 있으니 이를 확인해서 각 센서가 어떤 핀에 알맞는지 판단할 수 있어야 겠다. 
- 아날로그 가변저항 : 오른쪽으로 돌리면 값이 커진다. 왼쪽으로 돌리면 값이 작아진다. 그래프로 보여주니 확인이 용이하다.
- OLED : 오늘의 가장 큰 수확. 좀비게임, League of School 등의 게임을 하기 위해서는 반드시 디스플레이가 필요했다. 기본 LED 5x5로는 불가능했던 것들이 이제는 가능해졌다. 학교 동아리에서 잘 사용할 수 있을 듯 싶다. 이것 역시 케이스가 필수.
