# 코틀린 프로그래밍 공부

## 코틀린 변수
핵코교재 p.49~

코틀린의 변수는 **지역변수**다. 전역 변수의 개념이 없다. 값을 변경할 수 없는 이뮤터블과 값을 변경할 수 있는 뮤터블 변수로 나뉜다. 
불변 변수는 상수이며 `val` (value)로 표현하고 가변 변수는 `var`(variable)로 선언한다. 
```java
val a: Int = 7 //int타입의 a라는 변수에 7을 대입. 이뮤터블 변수.
val b = 10; //여기에서는 코틀린 컴파일러가 타입을 추룐하여 정수타입인 Int로 처리한다.
val c: Int
var d: Int = 0 // 뮤터블변수
```

## 코틀린 타입

코틀린의 기본 타입은 자바와 달리 모두 클래스로 정의되어 있다. 

정수타입

| 타입  | 크기(비트) | 허용값                                     |
|:------:|:----------:|:----------------------------------:|
| Byte  | 8          | -128 ~ 127                                 |
| Short | 16         | -32768 ~ 32767                             |
| Int   | 32         | -2147483648 ~ 2147483647                   |
| Long  | 64         | -9223372036854775808 ~ 9223372036854775807 |

실수타입

|  타입  | 크기(비트) |               허용값               |
|:------:|:----------:|:----------------------------------:|
|  Float |     32     | IEEE 754 부동 소수점 수(단 정밀도) |
| Double |     64     | IEEE 754 부동 소수점 수(배 정밀도) |


대부분의 자료형 변환은  `toInt()`, `toString()`, `toByte()`, `toFloat()` 와 같은 방식으로 할 수 있다. `String`을 `toInt()`로 바꿔서 출력하려고 하면 `NumberFormatException`이 발생한다. ~~당연하겠지~~


## 리터럴 
**리터럴**이라고 하는 것은 변수 선언 없이 지정한 값 자체를 의미한다. 

* 숫자 리터럴
  * Long 타입 : L을 붙인다. ex)100L
  * Float 타입 : F 또는 f를 붙인다. ex)3.07f
  * Double 타입 : 실수는 기본적으로 Double타입. 별도 표기가 없다.
  * 16진수 : 0x 또는 0X를 붙인다. ex)0XABFE, 0xcdfe
  * 2진수 : 0B 또는 0b를 붙인다.

* 문자열 리터럴

큰 따옴표 세 개를 이용해서 여러 줄의 왼쪽 여백과 줄바꿈이 모두 반영되어 우리가 입력한 그대로 저장된다.

문자열 리터럴에 변환 함수를 직접 붙여서 마음대로 바꿀 수도 있다.
```java
val c = "77".toInt()
val d = "123.5".toDouble()
println($c, $d)
```

$문자를 이용해서 문자열 템플릿을 만들면 문자열 결합에 사용할 수 있다. 변수를 가져다 쓸 경우에는 `$s1` 와 같이 $ 문자 뒤에  변수 이름을 붙이면 되지만, 표현식이나 배열을 사용할 때에는 `${s1.length}` 와 같이 중괄호 안에 넣어 주어야 한다.

```java
val count = 77
val s1 = "카운트 = $count"
val s2 = "$s1 의 길이는 ${s1.length}"
println(s1) // 카운트 = 77
println(s2) // 카운트 = 77 의 길이는 8
```
